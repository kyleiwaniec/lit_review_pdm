Two Paths to Physics-Informed Artificial Intelligence

Core Philosophy: How Physics is Integrated

Hamiltonian Neural Networks (HNNs)

"Hard Constraint" via Architecture

Physics is built into the network's structure. The architecture itself enforces conservation laws.

Input: State (q, p)
↓
NN Learns Scalar Hamiltonian (H)
↓
Dynamics are DERIVED from ∂H
↓
Output: Physically-Consistent Dynamics

Physics-Informed Neural Networks (PINNs)

"Soft Constraint" via Loss Function

Physics is enforced through penalties in the loss function. The network learns to satisfy the equations.

Input: Coords (x, t)
↓
NN Learns Solution u(x,t)
↓
Loss = L_eqn + L_bnd + L_ini + ...
↓
Output: Approximated Solution

Architectural Showdown

Feature Hamiltonian NN (HNN) Physics-Informed NN (PINN)
Physics Integration "Hard Constraint": Embedded in network architecture via Hamilton's equations. "Soft Constraint": Enforced as residual terms in the loss function.
Primary NN Output A single scalar value: the Hamiltonian (Energy). The solution function itself, e.g., u(x, t).
Role of Auto-Diff To derive system dynamics from the learned Hamiltonian's gradients. To calculate PDE residuals for the loss function.
Conservation Laws Inherently guaranteed and exactly preserved. Learned and approximated; not guaranteed.
Key Strength Exceptional long-term stability and physical fidelity. High flexibility to model a wide range of physical systems.
Key Limitation Restricted to conservative systems (no friction, etc.). Requires careful tuning of loss weights; physics is approximated.

Application Spectrum

The choice between HNNs and PINNs depends heavily on the problem's nature. HNNs are specialized for systems where conservation is key, while PINNs are general-purpose solvers for a broader class of differential equations.
